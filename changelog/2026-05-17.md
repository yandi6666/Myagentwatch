## B7 🔴 `_persist_data()` 数据持久化完全失效

**根因**: 第 2 层加固（5月15日）将 `_persist_data` 中 `conn = get_connection()` 改为
`with database() as conn:`，但只缩进了 `_ensure_source()` 3 行。第 208 行开始的
所有 `conn.execute()` / `conn.commit()` 缩进与 `with` 同级，连接已关闭后仍在调用。

**影响**: sessions、messages、parts、tool_calls、activity_log 全部无法写入。
**这是加固引入的 regression。**

**额外问题**: `conn.commit()` 缩进在 `for p` 循环内（每行 part 提交一次），应在循环外。

**修复**: `collector.py:208-308` 整体缩进 +4 空格入 `with` 块，`conn.commit()` 提到 `for p` 外。

---

## B12 🟡 `_daily_cleanup()` / `_dedup_agents()` 连接泄漏

**根因**: app.py 中两个清理函数仍用旧的 `conn = get_connection()` + `conn.close()` 模式，
如果 `conn.execute()` 抛异常，`conn.close()` 无法执行。其他模块（alerting、collector）
已迁移到 `database()` context manager，这两个被遗漏。

**修复**: 改为 `with database() as conn:` 包裹。

---

## B11 🟡 `init_db()` 连接泄漏

**根因**: `db.py:init_db()` 返回 `Connection` 但 `app.py:60` 不接收也不关闭。

**修复**: `init_db()` 内部 `conn.close()` 后不返回连接。

---

## B10 🟡 `waitress` 缺失

**根因**: `app.py:183` 尝试 `from waitress import serve` 作为生产 WSGI 服务器，
但 `requirements.txt` 未列出。

**修复**: `requirements.txt` 追加 `waitress>=3.0`。

## B8 ✅ 撤销 — `cache_hit_pct` 语义分析后保留

审计报告认为公式 `cache_read / (cache_read + cache_write)` 错误。
核实后结论：token_records 表无 `cache_miss` 字段，`cache_read` / `cache_write`
是 LLM API token 级缓存数据，不是请求级 hit/miss。当前公式是 token 维度上的正确度量。
不做修改，后续可考虑 metric 改名为 `cache_read_ratio` 以消除歧义。

---

## B13 🔴 Windows: `psutil.disk_usage("/")` 崩溃

**根因**: `psutil.disk_usage("/")` 在 Windows 上抛出 `FileNotFoundError`，Windows
无 Unix 根路径 `/`，需传驱动器号（如 `C:\`）。

**影响**:
- `myagentwatch/sources/system.py:32` → `collector.get_health()` 崩溃
- `routes/api.py:67` → `/api/health` 端点 500

**修复**: 两处改为 `psutil.disk_usage(os.path.abspath(os.sep))`，自动适配
Windows (`C:\`) 和 Linux (`/`)。`routes/api.py` 补充 `import os`（函数作用域内缺失）。

**验证**: Windows 实测 `/api/health` 返回 200，CPU/内存/磁盘数据正常。

---

## B14 🟡 Windows: `socketio.wsgi_app` 不存在

**根因**: 新版 Flask-SocketIO (5.4+) 移除了 `wsgi_app` 属性。`app.py:185` 仅捕获
`ImportError`，`AttributeError` 未被处理，导致应用启动失败。

**修复**: `except ImportError` → `except (ImportError, AttributeError)`，
graceful fallback 到 `socketio.run()`。
