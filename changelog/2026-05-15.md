# 2026-05-15 — Bug 修复 (5个关键Bug)

> 全面审计后修复，详情见 root cause analysis

## B1 🔴 通用适配器从未注册

**根因**: `sources/__init__.py` 未导入 `sqlite_agent` 和 `log_file` 模块，`@register_source` 装饰器从未执行。

**修复**: `sources/__init__.py` 追加 2 行 import:
```python
from . import log_file      # registers: log_file
from . import sqlite_agent  # registers: sqlite_agent
```

## B2 🔴 拓扑 SQL 缺空格导致语法错误

**根因**: `queries.py:115-116` 两行字符串拼接没有空格，产生 `''GROUP BY` 无效 SQL。

**修复**: 第 115 行末尾字符后加空格:
```python
"WHERE s1.agent_id != '' AND s2.agent_id != '' "  # ← 末尾加空格
```

## B3 🔴 前端死代码 + dagre 依赖缺失

**根因**:
- `flow.js` 引用未加载的 `dagre` 库，与 `node-detail.js` 函数名冲突，且 `index.html` 从未加载
- `dashboard.js` 是旧版实现，重新声明 `const API` 会与 `app.js` 冲突

**修复**: 删除 `static/js/flow.js` 和 `static/js/dashboard.js`（死代码）

## B4 🔴 告警引擎数据库连接泄漏

**根因**: `alerting.py:39-41` 无 try/finally，`_get_metric_value()` 抛异常时 `conn.close()` 不执行。

**修复**: 加 try/finally 包裹:
```python
conn = get_connection()
try:
    current_value = self._get_metric_value(conn, metric)
finally:
    conn.close()
```

## B5 🔴 外键约束仅在 init_db 启用

**根因**: `db.py` 的 `get_connection()` 不设置 `PRAGMA foreign_keys=ON`，运行时外键完整性被静默忽略。

**修复**: `get_connection()` 追加:
```python
conn.execute("PRAGMA foreign_keys=ON")
```
