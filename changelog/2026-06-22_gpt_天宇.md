## Claude Code 运行中却显示离线修复

**Agent：GPT**  
**用户：天宇**  
**时间：2026-06-22**

### 问题

页面中 Claude Code 明明有进程在运行，但集群拓扑视图显示：

```text
Claude Code 0在线 5离线
```

预期应该是：

```text
Claude Code 0在线 5等待
```

### 原因

旧状态机只看：

- heartbeat
- activity_log
- last_seen_time

没有把 P0-A/P0-B/P0-C 下沉到 `myagentwatch-cli` 的 `agent_processes` 进程采集结果作为运行时存活信号。

因此 Claude Code 进程虽然存在，但如果日志活动或心跳超过超时阈值，就会被判定为：

```text
activity_lost -> offline
```

### 修复

修改文件：

```text
C:\Users\天宇\Desktop\claude-win32-x64\myagentwatch\myagentwatch\myagentwatch\collector.py
```

新增逻辑：

- 从 `agent_processes` 读取最近的 `detected_role`。
- 将 `claude-code` 进程映射到 Claude Code 数据源。
- 如果进程近期存在，但没有新的工作/心跳，则状态回退为 `idle`。
- `idle` 在前端显示为“等待”，而不是“离线”。
- 如果进程不存在并且信号超时，仍然保持 `offline`。

新增状态原因：

```text
process_detected_waiting
process_detected_source_disconnected
```

### 验证

数据库中确认近期存在：

```text

process_name = claude
```
