## 心跳与状态机修复

### 心跳状态不再被丢弃

修复前：

- `/api/heartbeat` 接收了 `status / metadata / model_id`
- 但 `flush_heartbeats()` 写库时只更新心跳时间，状态信息被丢弃

修复后：

- 写入 `status`
- 写入 `status_since`
- 写入 `model_id`
- metadata 中保存：
  - `heartbeat_status`
  - `heartbeat_metadata`
  - `heartbeat_auth_user`

### 状态机尊重 Agent 自报状态

- 对近期有心跳的 Agent，如果自报 `working / blocked / idle`，采集器不会立即覆盖成 `active`。
- `cli` 类型加入无需模型号的类型列表，避免 CLI 接入器因为模型号逻辑被误判。

## myagentwatch-cli 增强

### 新增 task 命令

新增 CLI 命令：

| 命令 | 功能 |
|---|---|
| `myaw task list` | 查看任务 |
| `myaw task create <title>` | 创建任务 |
| `myaw task start <id>` | 标记运行中 |
| `myaw task complete <id>` | 标记完成 |
| `myaw task fail <id>` | 标记失败 |
| `myaw task cancel <id>` | 取消任务 |
| `myaw task queue <id>` | 重新排队 |

底层新增：

- `client.patch()`

### 心跳默认读取配置

`heartbeat` 命令现在在未传 `--agent-id` 时会自动读取 `config.json` 中的 `agent_id`。

这样后台 daemon 不需要再传带空格/冒号的长 Agent ID，避免 Windows 参数拆分问题。
