# 2026-05-16 — 拓扑图节点拖动失效

## B6 🔴 仪表盘拓扑图节点无法拖动 / 拖完自动复原

节点拖动有 4 层问题，缺一不可。逐层修复如下。

---

### 问题1 — zoom 与 drag 事件冲突

`topology.js:23` SVG 上挂了 `d3.zoom()`，每个 `<g>` 节点上挂了 `d3.drag()`。
D3 v7 中 drag 默认不阻止 zoom，mousedown 时两个处理器同时收到事件，父级 zoom 优先，drag 失效。

**修复**: zoom 加 filter，对 `.topo-node` 区域不触发：
```javascript
svg.call(d3.zoom()
    .scaleExtent([0.3, 3])
    .filter(event => !event.target.closest('.topo-node'))
    .on('zoom', e => g.attr('transform', e.transform)));
```

### 问题2 — snapshot 刷新冲掉位置

每 2 秒 WebSocket 快照，`_draw()` 从 `topo.nodes` 创建**全新数据对象** (`{...n}`)，没有 `x`/`y` 坐标。渲染器用 undefined 重绘 → 节点弹回。

**修复**: `_updateSimulation()` 开头加位置传承，旧 sim 坐标传给新数据。

### 问题3 — filter 被子元素击穿

`event.target` 是鼠标实际点击的 DOM 元素。节点结构：
```html
<g class="topo-node">          ← 有这个 class
  <circle />                   ← 没有这个 class，filter 匹配失败!
  <text />                     ← 没有这个 class，filter 匹配失败!
</g>
```
点 circle/text 时 filter 认不出，zoom 照样激活。`classList.contains` 只查自身，不查祖先。

**修复**: `classList.contains` → `closest()`，沿祖先链上查。

### 问题4 — 拖拽位置未回传到 sim

`_updateSimulation()` 的 `else` 分支（节点集不变时）只更新属性，没调 `sim.nodes(nData)`。
导致 sim 一直引用旧数据对象，下一次 carry-forward 读到的是上次的位置。

时间线：
1. 拖拽修改了 DOM 绑定的 `nData` 对象
2. 下次快照 carry-forward 从 `sim.nodes()` 读位置 → 读到的还是旧值
3. 旧值写回 → 节点复原

**修复**: `else` 分支末尾加 `sim.nodes(nData)`，让 sim 指向最新数据，下一轮 carry-forward 读到正确位置。

### 总结

| # | 问题 | 修复 | 位置 |
|---|------|------|------|
| 1 | zoom & drag 同时收到事件 | zoom filter 排除 `.topo-node` | `topology.js:23` |
| 2 | snapshot 创建无坐标新对象 | carry-forward 位置传承 | `_updateSimulation()` 开头 |
| 3 | filter 不认子元素 | `classList.contains` → `closest()` | `topology.js:24` |
| 4 | sim 引用未同步 | `else` 加 `sim.nodes(nData)` | `_updateSimulation()` else 末尾 |
